# 🎵 MediaPlayer — GitHub Pages Music Player

A fully-featured, self-hosted music and video player that runs entirely on GitHub Pages. No server, no backend, no cost.

---

## ✨ Features

- 🎵 Auto-loads all audio/video files from your `music/` folder
- 📋 Custom playlists — create, name, and manage multiple playlists
- 🖱️ Right-click any track to add/remove from playlists or favorites
- ♥ Favorites library
- 🕐 Recently Played history
- ⬇️ Download any track directly from the player
- 🔀 Drag-and-drop track reordering
- 💾 Export/Import playlist order as JSON
- ▶ Full playback controls: Play, Pause, Stop, Next, Prev, Shuffle, Repeat
- 🔊 Volume control, Mute, Playback Speed (0.5× – 2×)
- 🎚️ Equalizer presets: Flat, Bass+, Treble+, Vocal, Electronic
- 🌙 Dark / Light theme toggle
- ⏱️ Sleep Timer with Enable/Disable toggle (won't stop music while disabled)
- ⊡ Mini Player — floating bar so you can scroll while music plays
- 🔁 Crossfade between tracks
- 🎬 Video support with Audio-Only toggle (saves CPU/battery)
- 🌐 Multi-repo support — load music from multiple GitHub repos (bypass 1 GB limit)
- ⌨️ Keyboard shortcuts
- 📱 Responsive — works on mobile and desktop
- ⬇️ YouTube download via GitHub Issues + yt-dlp

## 🚀 Quick Setup

### 1. Fork or upload this repo to GitHub

Upload all files from the zip into a new GitHub repository.

### 2. Enable GitHub Pages

Go to: **Repo → Settings → Pages → Source → GitHub Actions**

### 3. Trigger first deploy

Edit any file and commit (e.g. add a space to README.md), or go to **Actions → Deploy Music Player → Run workflow**.

### 4. Your site is live!

```
https://YOUR_USERNAME.github.io/YOUR_REPO_NAME
```

## 🎵 Adding Music

### Option A — Upload directly on GitHub
1. Go to your repo → `music/` folder
2. Click **Add file → Upload files**
3. Drag your `.mp3`, `.mp4`, `.wav` etc. files
4. Commit — the site rebuilds in ~1 minute

### Option B — Download from YouTube (yt-dlp)
1. Go to your repo → **Settings → Secrets and variables → Actions**
2. Create a secret called `YOUTUBE_COOKIES` (paste your YouTube cookies from the browser extension **"Get cookies.txt LOCALLY"**)
3. Open a **GitHub Issue** with the title:

**Single song:**
```
ytdlp: https://youtu.be/VIDEO_ID
```

**Full playlist:**
```
ytdlp: https://youtube.com/playlist?list=PLAYLIST_ID
```

**Specific range (songs 1–60):**
```
ytdlp: https://youtube.com/playlist?list=PLAYLIST_ID start:1 end:60
```

The song downloads as MP3, commits to `music/`, and the site rebuilds automatically. The issue gets closed with a confirmation comment.

## 🌐 Multi-Repo (More Than 1 GB of Music)

Each GitHub repo can hold ~1 GB. To go beyond that:

1. Create a second GitHub repo (e.g. `my-music-2`)
2. Upload music files to its `music/` folder (with the same workflow)
3. In the player, click **⚙ Settings → Add Repo** → enter `username/repo-name`

The player combines all repos into one playlist.

## 🖱️ How to Use the Player

| Action | How |
|--------|-----|
| Play a track | Click it in the playlist |
| Add to playlist | Right-click → Add to playlist |
| Add to favorites | Right-click → Add to Favorites, or press **F** |
| Remove from playlist | Right-click → Remove from this playlist |
| Reorder tracks | Drag the ⠿ grip handle |
| Save playlist order | Click ↓ in the playlist header |
| Load saved order | Click ↑ and select your `playlist.json` |
| Switch video to audio-only | Click **🔊 Audio Only** button on the video |
| Mini player | Click ⊡ in the header |
| Sleep timer | Click ⏱ in the header — toggle ON/OFF without losing your set time |
| Equalizer | Use the EQ buttons below the controls |

### Keyboard Shortcuts
| Key | Action |
|-----|--------|
| `Space` | Play / Pause |
| `→` | Next track |
| `←` | Previous track |
| `S` | Stop |
| `M` | Mute / Unmute |
| `F` | Toggle favorite |
| `Esc` | Close modals |

## 🔧 Forking to Another Account

1. Fork the repo
2. Go to forked repo → **Settings → Features → enable Issues**
3. Go to **Settings → Pages → Source → GitHub Actions**
4. Add your own `YOUTUBE_COOKIES` secret (Settings → Secrets → Actions)
5. Go to **Actions** tab → enable workflows
6. Run the **Deploy** workflow manually once

## 📁 File Structure

```
your-repo/
├── index.html           ← Player UI
├── style.css            ← All styles (dark + light theme)
├── script.js            ← All player logic
├── README.md            ← This file
├── music/
│   ├── index.json       ← Auto-generated by workflow (do not edit)
│   ├── Song1.mp3
│   ├── Song2.mp3
│   └── Video.mp4
└── .github/
    └── workflows/
        ├── deploy.yml   ← Auto-deploys site + regenerates manifest
        └── ytdlp.yml    ← Downloads YouTube audio via Issues
```

## ⚠️ Limits

| Thing | Limit |
|-------|-------|
| Repo size | 1 GB (use multi-repo to go beyond) |
| Monthly bandwidth | ~100 GB (GitHub soft limit) |
| Single file | < 100 MB recommended |
| Large video files | May stutter (GitHub Pages has no range-request support) |
| yt-dlp job time | 6 hours max per GitHub Actions run |
